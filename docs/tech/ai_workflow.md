# noche AI workflow

## 1. 角色分工

- `Antigravity`：负责任务编排、拆解、验收和阶段推进
- `Codex`：负责工程、底层、数据、配置与可编译骨架，不负责正式页面实现
- `Gemini`：负责页面与 UI 组件实现

## 2. 写入边界

- 同一时间、同一目录只允许一个 AI 写入
- 一个 AI 正在改动某个目录时，其他 AI 只能读，不能并发写
- 跨目录协作前要先明确负责范围，避免覆盖彼此改动

## 3. 推荐协作方式

### 3.1 Antigravity

- 产出任务拆解
- 指定目录负责人
- 在阶段结束时做验收

### 3.2 Codex

- 初始化工程
- 补齐配置
- 维护 domain / data / shared / app 骨架
- 处理依赖、构建、验证

### 3.3 Gemini

- 基于已稳定的接口与路由做页面实现
- 负责视觉层和交互层组件
- 不直接改底层 schema、store 骨架和工程配置

## 4. 当前协作规则

- 先由 Codex 把底层工程和可编译骨架稳定下来
- 再由 Gemini 基于稳定接口实现页面
- Antigravity 负责验收规则是否与产品文档一致
- 每完成一个阶段需生成 handoff / checkpoint 文档
- 稳定阶段需本地 commit，防止上下文丢失

## 5. 禁止事项

- Codex 不直接承担正式页面设计任务
- Gemini 不在未约定的情况下改动底层工程配置
- 多个 AI 不得同时在同一路径下提交写入
- 禁止将明文数据库密码写入将要提交的文件
