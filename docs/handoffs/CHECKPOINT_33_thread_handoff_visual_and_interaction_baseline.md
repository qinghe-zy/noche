# CHECKPOINT 33 - Thread Handoff Visual And Interaction Baseline

主验收设备：`Redmi Note 11T Pro+`

## 1. 当前结论

当前仓库已经从“功能先通”进入“视觉与交互双收口”阶段，且主链页面不再只是占位壳：

- `Home` 已收成中央纸页主入口
- `Mailbox` 已改成双层切换 + 模块化内容区
- `Calendar` 已改成“点日期后在当前页打开当天小邮箱”
- `Editor` 已开始按 `docs/stitch/editor_immersive_stationery` 收成沉浸式信纸
- `Profile` 仍可继续做最终视觉统一，但已不是 TODO 壳页

当前最重要的不是继续堆新功能，而是：

1. 把主链页面的视觉稿进一步贴齐
2. 在 `Redmi Note 11T Pro+` 的真实长屏上验证排版
3. 收掉少量仍然偏“网页表单感”的局部

## 2. 当前干活方式

后续新线程请延续以下方式，不要切回“大改大猜”：

### 2.1 真相源优先级

1. `docs/tech/**`
2. `docs/stitch/**`
3. 当前已通过的测试
4. 页面现有稳定 store / domain / facade 语义

不要让页面自己重新发明产品规则。

### 2.2 改动节奏

每轮默认按下面节奏推进：

1. 先确认参考稿和现状差异
2. 先补窄红灯测试锁目标骨架或规则
3. 再改页面 / store / helper
4. 最后统一验证：
   - `pnpm test:unit`
   - `pnpm type-check`
   - `pnpm build:h5`

### 2.3 页面改法

- 视觉页优先“承接并重排”，不要推倒重写业务逻辑
- 新规则优先落在 `store / domain / facade / repository`
- 页面层只做：
  - 版式
  - 状态编排
  - 轻反馈

### 2.4 视觉验收法

- 默认用本地 H5 截图做中间验收
- Playwright MCP 当前仍可能受 `C:\Windows\System32\.playwright-mcp` 权限影响
- 当前稳定截图方式是：
  - 本地启动 `dev:h5`
  - 使用 Edge headless 抓图到 `artifacts/shots/latest/*.png`

### 2.5 编辑页特殊规则

`Editor` 这页优先守交互语义，不要为了视觉稿破坏：

- 自动静默暂存 = 草稿层
- 右上小信封 = 正式保存 / 正式封存
- 阅读态右上 = 续写
- 不允许点正文直接进入编辑态

## 3. 当前验收标准

后续新线程请默认用这套标准验收：

### 3.1 页面级验收

- 结构和层级尽量贴近 `docs/stitch/**`
- 不保留原型腔英文文案、假数据腔、工具味按钮
- 不让系统导航栏叠在页面上
- 页面首屏主视线要稳定，不凌乱、不漂浮

### 3.2 设备级验收

以 `Redmi Note 11T Pro+` 为主验收设备，优先确保：

- 空状态一屏能看完
- 键盘弹起后输入位置不被遮挡
- 信纸正文和横线持续对齐
- 背景 / 纸面 / 操作按钮比例不溢出

### 3.3 主链级验收

- `Home -> Diary`
- `Diary -> formal save -> read mode`
- `Mailbox -> read`
- `Calendar -> day mailbox -> read`
- `read mode -> continue write`

任何视觉收口都不能破坏这条链。

## 4. 已经明确的标准化规则

### 4.1 书写页字号标准

已写入仓库规则：

- 所有新增书写页面正文默认字号统一使用 `18px`

位置：

- `AGENTS.md`

### 4.2 日历交互标准

- 月历层只保留：
  - 日期数字
  - 一个统一小点
  - 今天高亮
  - 选中态
- 所有日期都能点
- 点击后都在当前页打开“小邮箱”
- 差异只体现在小邮箱的列表态 / 空态

### 4.3 信箱分组标准

- 一级：
  - `纪实`
  - `寄远`
- 二级：
  - `纪实` -> `日记 / 随笔`
  - `寄远` -> `待启 / 已启`

右下角写作按钮跟随当前二级语义走。
