# noche 产品规格 v2

版本：v2  
状态：可进入开发前定稿  
适用阶段：Android 单端、本地优先、持续迭代

---

## 1. 产品定位

一个以“写给自己”为核心隐喻的私人记录 App，帮助用户写日记、写随笔、给未来写信，并通过信箱与日历回看自己的时间痕迹。

### 核心隐喻

- 写内容 = 写一封信给自己
- 点信封 = 把这张纸收起来 / 正式封存
- 回看内容 = 打开私人信箱
- 未来信 = 封存到指定日期再开启的信

### 当前平台范围

- 当前阶段只做：Android 手机端
- 当前阶段不做：Windows / 平板 / iOS / Web 正式版本

### 产品气质

- 极简
- 温暖
- 私人化
- 纸张感
- 信件感
- 安静、克制、有轻仪式感

---

## 2. 当前范围与非目标

### 当前核心能力

1. 写日记
2. 写随笔
3. 给未来写封信
4. 查看信箱
5. 按日历跳转记录
6. 个人设置与隐私锁

### 当前明确不做

- 不做复杂富文本编辑器
- 不做多人协作
- 不做社交分享
- 不做复杂云同步
- 不做企业级后端
- 不做复杂统计分析
- 不做复杂提醒系统
- 不做图片/音频能力的正式功能，但要为未来预留结构

## 3. 内容模型与业务定义

### 三种核心内容类型

#### A. 日记 Diary

- 按天唯一
- 每个自然日最多 1 篇日记
- 可多次追加和修改
- 可通过日历补写历史日期

#### B. 随笔 Jotting

- 随时记录灵感
- 一天可以有无数多条随笔
- 不支持通过日历补写历史随笔
- 允许存在一个“活跃随笔草稿槽”

#### C. 未来信 Future Letter

- 写给未来的自己
- 必须指定未来日期
- 封存后在指定日期本地时间 00:00 解锁
- 正式封存后不可编辑、不可改日期，只能等待开启或销毁

### 页面内展示口径

- 日记、随笔、已解锁未来信属于“往日内容”
- 信箱标签页命名统一为：
  - 往日信件
  - 待启之信

## 4. 页面结构

### 4.1 首页 Home

作用：进入核心写作与归档空间。

入口：

- 中间主入口：打开今日信纸 / 写今日内容
- 下方入口：日记、未来信、邮箱
- 右上角头像：个人中心

说明：

- 首页是入口场，不是信息中心
- 不放底部导航
- 不放 Recent feed
- 不放悬浮按钮

### 4.2 通用编辑页 Editor

作用：统一承接日记、随笔、未来信的编辑态与阅读态。

模式：

- 日记模式
- 随笔模式
- 未来信模式
- 阅读态（只读）

### 4.3 信箱页 Mailbox

作用：回看过去内容与查看未开启未来信。

标签页：

- 往日信件
- 待启之信

入口：

- 首页点“邮箱”进入
- 右上角日历图标进入日历页

### 4.4 日历页 Calendar

作用：按日期查看是否有记录，并跳转到当日内容。

入口：

- 从信箱页右上角日历图标进入

退出：

- 左上返回回信箱页

右上角：

- 不再跳新页面
- 右上角图标定义为“回到今天”

### 4.5 个人中心 Profile

作用：个人信息、外观、隐私锁、备份、关于。

入口：

- 首页右上角头像

## 5. 核心交互流

### 5.1 首页

- 点中间主卡 -> 进入当天写作入口
- 点“日记” -> 进入日记相关入口 / 当日日记
- 点“未来信” -> 进入未来信编辑模式
- 点“邮箱” -> 进入信箱页
- 点头像 -> 进入个人中心

### 5.2 编辑页

- 默认进入即聚焦写作
- 输入过程中自动静默暂存草稿
- 点右上角信封 = 正式收好 / 保存
- 正式保存后，当前页由编辑态切为阅读态
- 阅读态右上角提供“续写”入口，进入编辑态
- 不允许点正文直接进入编辑态

### 5.3 信箱页

- 在“往日信件”查看已保存内容
- 在“待启之信”查看未开启未来信
- 点内容卡片进入阅读态
- 点右上角日历图标进入日历页

### 5.4 日历页

- 有记录日期带点/线标记
- 点有记录日期：
  - 若该日期只有 1 条内容，直接进入该内容阅读态
  - 若该日期有多条内容，进入“当天归档列表页”
- 点无记录历史日期：固定新建该日期的日记
- 点无记录且是今天：进入今日日记新建/续写

### 5.5 未来信

- 进入未来信编辑模式
- 编写内容
- 点信封时，如果还没选日期，则弹出底部半屏日期选择器
- 选定日期后立刻封存
- 未到日期不可查看全文
- 到指定日期 00:00 自动解锁

## 6. 关键业务规则（整合版）

### 6.1 recordDate 与排序

- `recordDate` 在打开纸张时锁定
- 信箱排序按 `recordDate`，不按 `updatedAt`

### 6.2 保存与草稿

- 空白内容不能正式保存
- 后台 / 失焦时需要静默暂存
- 草稿与正式条目分离，草稿先行、正式保存后生成或更新条目
- 草稿按类型隔离，日记草稿按日期分槽

### 6.3 未来信

- 最早只能选明天
- 未到解锁日时处于锁定
- 到达解锁日后进入可打开状态

### 6.4 删除

- 删除统一走 `destroyEntry`
- `destroyEntry` 预留资源清理 Hook

### 6.5 日历

- 日历补写只能补日记，不能补写随笔或信件
