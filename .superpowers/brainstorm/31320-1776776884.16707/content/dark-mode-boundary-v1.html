深色模式全面升级：边界确认

目标是重做暗色体验，但不扰动浅色模式；先确认范围，再谈页面视觉。

Boundary Map · 深色模式专属改造
In Scope

只在 theme-dark 生效

  • 主页按 chisu-v2.jsx 复刻：底部导航、今日一问、最近随笔、邮箱入口
  • 邮箱 / 日历 / 条目阅读 / 随笔 / 未来信页面跟随 chisu-v2 的暗色壳
  • 新增「档案馆」页面，交互参考 archives.jsx
  • 档案馆每日一问：每天一个问题，回答保存为私人日记式记录
  • 一年后展示「今天一年前你在想什么」
Protected

浅色模式保持原样

  • 不改浅色页面布局和视觉 token
  • 不改现有日记 / 随笔 / 未来信的保存逻辑
  • 不破坏已有草稿、邮箱、日历和阅读入口
  • 不重写无关 store / repository
  • 必要数据扩展要可迁移、可回退、可测试
Recommended Data Boundary

档案馆不要新建一套孤立日记系统；推荐新增一个 archive-question/answer 的小领域模型， 保存问题、回答、日期，再把展示和“一年前回看”独立在 archive feature 内。 这样暗色体验可以重做，原来的 Entry 业务仍保持稳定。
